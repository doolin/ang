This section builds on the isolate scope section covered previously. The notion is that having an isolate scope is rendered much more useful when able to bind data from outside the directive scope to within the isolate scope.

This is accomplished by passing one of the following three alaises in the object:

  1. @ or @attr: Local Scope Property binds a local scope (string) to the value of the DOME attribute: the outer scope will be available inside the directive scope.
  2. = or =attr: Bi-directional binding the local property reflects change in the parent property.
  3. & or &attr: Parent execution binding point to a function wrapper passed in where the key is the name on the argument and the value being the argument to pass.
